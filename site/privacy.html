Privacy policy

PALS Engine QA Auditor

Effective date: 2026-05-05

PALS is designed for local QA and accessibility audits of pages the user owns, controls, or is authorized to test.

Data Handling

The Chrome extension reads the current page structure only when the user runs Quick Audit or Live Capture. Scan artifacts are generated locally in the browser and downloaded by the user as Markdown or JSON.

The extension may process visible layout, DOM element metadata, pointer coordinates, page title, URL, field labels, button names, hover changes, and QA/accessibility findings to create the local audit report.

Credentials And Sensitive Content

PALS is not designed to collect credentials, payment data, health data, private messages, or other sensitive personal information. Users should run audits only on authorized pages and avoid capturing content outside the approved testing scope.

No Sale Or Advertising Use

PALS does not sell user data, does not use browsing activity for advertising, and does not transfer audit data to advertising platforms, data brokers, or unrelated third parties.

Remote Servers

The extension package does not send scanned page content to PALS servers and does not execute remotely hosted code. The public website and demo are static assets hosted for product education and pilot requests.

Downloads

When the user chooses to download a report, Chrome writes the selected Markdown or JSON file to the user's machine using the browser download flow.

Limited Use Statement

The use of information received from Google APIs will adhere to the Chrome Web Store User Data Policy, including the Limited Use requirements.

Contact

For privacy or support requests, use the pilot/contact flow on the PALS Engine website.
